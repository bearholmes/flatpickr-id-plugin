Transfer ID and accessibility attributes to flatpickr inputs

View on GitHub

Basic Example Default

What's happening?

The id="basic-datepicker" and title="Basic Datepicker" attributes are automatically transferred from the original input to flatpickr's generated input element. 

flatpickr('#basic-datepicker', {
  plugins: [idPlugin()]
});

Accessibility Example ARIA

Accessibility Features

ARIA attributes like aria-label and aria-labelledby are preserved, ensuring screen readers work correctly. 

flatpickr('#birth-date', {
  plugins: [idPlugin()],
  dateFormat: 'm/d/Y'
});

Custom Configuration Advanced

Configuration Options

  • delay: Custom delay before transferring attributes (default: 10ms)
  • attributes: Custom list of attributes to transfer
  • debug: Enable console logging for debugging 
flatpickr('#custom-datepicker', {
  plugins: [idPlugin({
    delay: 20,
    attributes: ['id', 'title', 'data-custom'],
    debug: true
  })]
});

Date Range Example Range

flatpickr('#date-range', {
  mode: 'range',
  plugins: [idPlugin()]
});

Features

  • Automatically transfers id, title, aria-label, aria-labelledby
  • Works with altInput and mobileInput modes
  • Configurable delay and attribute list
  • TypeScript support with full type definitions
  • Comprehensive error handling
  • Optional debug logging
  • Zero dependencies (except flatpickr)
  • Lightweight (~2KB minified)
